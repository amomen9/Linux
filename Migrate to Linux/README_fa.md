# مهاجرت به لینوکس — انتقال نرم‌افزار، تنظیمات و درایور

## مقدمه

مهاجرت از ویندوز به لینوکس معمولاً یعنی ساعت‌ها وقت گذاشتن برای اینکه «اصلاً چه برنامه‌هایی نصب داشتم؟»، حدس‌زدن معادل‌های لینوکسی، کپی‌کردن دستی تنظیمات، و دعا برای اینکه سخت‌افزار کار کند. **این پروژه همهٔ این کارها را برای شما انجام می‌دهد.**

این یک جعبه‌ابزار کامل و **خودکار برای مهاجرت ویندوز → لینوکس** است که:

- **همهٔ برنامه‌های نصب‌شده روی کامپیوتر ویندوزی‌تان را فهرست می‌کند** و هرکدام را برای لینوکس رتبه‌بندی می‌کند: آیا نسخهٔ بومی (native) دارد، آیا جایگزین خوبی دارد، آن جایگزین چقدر خوب است، آیا رایگان است، و آیا ارزش نصب دارد.
- **همهٔ برنامه‌های ماندگار را به‌صورت خودکار روی لینوکس نصب می‌کند** — بسته‌های بومی، Flatpak، ایمیج‌های Docker، فایل‌های `.deb` سازنده، و حتی خودِ برنامه‌های ویندوزی را زیر Wine آنجا که بهترین گزینه است.
- **تنظیمات رایج ویندوز شما را استخراج می‌کند** (رفتار دکمهٔ روشن/خاموش و بستن درب لپ‌تاپ، رزولوشن و مقیاس نمایش، چیدمان صفحه‌کلید، تله‌متری، به‌روزرسانی خودکار) و **آن‌ها را برای همهٔ کاربران روی لینوکس دوباره اعمال می‌کند**.
- **همهٔ درایورهای دستگاه را فهرست می‌کند** و **معادل‌های لینوکسی را نصب می‌کند**، و میان‌افزار (firmware) را مستقیماً از سازنده از طریق `fwupd`/LVFS دریافت می‌کند.

همهٔ محتوای این پوشه از پیش برای **همین دستگاه** تولید شده است؛ بنابراین در بیشتر موارد کافی است آن را روی دستگاه لینوکس جدیدتان **کپی کنید و یک فرمان اجرا کنید** — بدون نیاز به هوش مصنوعی و بدون تحقیق دستی.

```bash
cd "Migrate to Linux/Linux Mint (Ubuntu)"
sudo ./execute_all.sh        # تنظیمات → برنامه‌ها → درایورها، با سیاست ادامه‌در‌صورت‌خطا
```

## نحوهٔ استفاده — گردش‌کارها (Workflows)

> **برای استفادهٔ معمول به هیچ عامل هوش مصنوعی نیاز ندارید.** اسکریپت‌ها و فایل‌های
> CSV از‌پیش‌تولیدشده در این پوشه آمادهٔ اجرا هستند. کافی است گردش‌کارهای زیر را دنبال کنید.
>
> یک عامل هوش مصنوعی (که فایل [`instructions.txt`](instructions.txt) را می‌خواند) فقط
> برای چند کار **اختیاری و یک‌باره** که مخصوص حالتِ *شما* است لازم می‌شود:
> - **بازتولید گزارش‌ها/نصب‌کننده از صفر** با دادهٔ تازه و تحقیق‌شده برای یک **کامپیوتر ویندوزی متفاوت** از آنچه اینجا ثبت شده است.
> - **افزودن یک سیستم‌عامل/توزیع هدف جدید** (مثلاً Fedora یا Arch) — تولید مجموعهٔ تازه‌ای از اسکریپت‌های نصب برای آن.
> - **به‌روزرسانی رتبه‌بندی برنامه‌ها و نگاشت درایورها** با جدیدترین تحقیق وب.
>
> تولید گزارش‌ها روی ویندوز (مرحلهٔ ۱ از گردش‌کارهای ۱ تا ۳) و اجرای نصب‌کننده‌ها روی
> لینوکس هرگز به عامل هوش مصنوعی نیاز ندارند.

این کار دو مرحله دارد:

1. **روی ویندوز** — سه اسکریپت PowerShell را اجرا کنید تا گزارش‌های CSV (دوباره) تولید شوند. *اگر همان دستگاهی را مهاجرت می‌دهید که این فایل‌ها روی آن تولید شده‌اند این مرحله را رد کنید — فایل‌های CSV از قبل اینجا هستند.*
2. **روی لینوکس** — این پوشهٔ `Migrate to Linux/` را کپی کنید و نصب‌کننده‌ها را با دسترسی root اجرا کنید.

### گردش‌کار ۰ — اجرای همه‌چیز با یک فرمان (توصیه‌شده)

روی دستگاه لینوکس مقصد، پس از کپی‌کردن فایل‌های CSV تولیدشده داخل
`Migrate to Linux/`، هماهنگ‌کنندهٔ واحد را با دسترسی root اجرا کنید:

```bash
cd "Migrate to Linux/Linux Mint (Ubuntu)"
sudo ./execute_all.sh
```

این فرمان سه اسکریپت مرحله‌ای را اجراپذیر می‌کند و آن‌ها را **به‌ترتیب — تنظیمات →
برنامه‌ها → درایورها — با سیاست ادامه‌در‌صورت‌خطا** اجرا می‌کند (خطا در یک مرحله ثبت
می‌شود اما هرگز جلوی بقیه را نمی‌گیرد). هر مرحله رابط کاربری تمیز و لاگ‌های خودش را
دارد؛ `execute_all.sh` در پایان یک خلاصهٔ OK/FAILED برای هر مرحله چاپ می‌کند و اگر
هر مرحله‌ای ناموفق باشد با کد خروج غیرصفر تمام می‌شود. هرگاه فقط یک مرحله را می‌خواهید
اجرا کنید، گردش‌کارهای جداگانهٔ زیر را ترجیح دهید.

### گردش‌کار ۱ — مهاجرت نرم‌افزارهای نصب‌شده

1. **تولید گزارش** *(ویندوز؛ اگر از قبل موجود است رد کنید)* — اسکریپت `detect_installed_windows_software.ps1` را روی کامپیوتر ویندوزی اجرا کنید.
2. **بازبینی** `installed_windows_software.csv` — به‌ویژه ستون‌های `Must be included on Linux` و `Can be synched to Linux alternative`.
3. **نصب روی لینوکس** — اسکریپت `install_must_have_software.sh` را با دسترسی root اجرا کنید.

### گردش‌کار ۲ — مهاجرت تنظیمات ویندوز

1. **استخراج تنظیمات** *(ویندوز؛ اگر از قبل موجود است رد کنید)* — اسکریپت `windows_settings_extract.ps1` را روی کامپیوتر ویندوزی اجرا کنید:

   ```powershell
   powershell -ExecutionPolicy Bypass -File windows_settings_extract.ps1
   ```

   این فرمان `windows_configs.csv` را تولید می‌کند.
2. **کپی** `windows_configs.csv` به پوشهٔ `Migrate to Linux/` روی دستگاه لینوکس.
3. **اعمال** — از داخل پوشهٔ توزیع، با دسترسی root اجرا کنید:

   ```bash
   cd "Migrate to Linux/Linux Mint (Ubuntu)"
   sudo ./apply_settings.sh
   ```

   این فرمان `../windows_configs.csv` را می‌خواند و اعمال می‌کند (**برای همهٔ کاربران** — یادداشت زیر را ببینید):

   - **برق (Power):** کنش بستن درب لپ‌تاپ (باتری و برق شهر) → `logind.conf`
   - **نمایش (Display):** رزولوشن → `xrandr`، مقیاس‌بندی → مقدار پیش‌فرض سراسری dconf
   - **صفحه‌کلید (Keyboard):** چیدمان → `localectl` + `setxkbmap`، مرجع میان‌برها
   - **تله‌متری (Telemetry):** غیرفعال‌کردن `whoopsie`، `apport`، `apt-daily.timer`، `motd-news.timer` و غیره
   - **مکان (Location):** غیرفعال‌کردن `geoclue`، تصادفی‌سازی MAC، مکان‌یابی GNOME (سراسری)
   - **صفحه (Screen):** زمان قفل/خاموشی صفحه → dconf سراسری (`"never"` قفل و خاموشی را غیرفعال می‌کند)
   - **به‌روزرسانی خودکار (Auto-update):** نصب `system_update.service` + `system_update.timer` از مخزن

   > **برای همهٔ کاربران اعمال می‌شود.** کلیدهای GNOME/میزکار (مقیاس‌بندی، مکان، زمان
   > قفل صفحه) به‌صورت **مقادیر پیش‌فرض سراسری dconf** در `/etc/dconf/db/local.d`
   > نوشته می‌شوند — نه با `gsettings set` به‌عنوان root (که فقط پروفایل خودِ root را
   > تغییر می‌دهد). به همراه تغییرات سراسری `logind`/`systemd`/`localectl`/APT، هر
   > تنظیم برای **همهٔ کاربران** دستگاه (فعلی و آینده) در ورود بعدی اعمال می‌شود.
   >

### گردش‌کار ۳ — مهاجرت درایورهای دستگاه

1. **تولید گزارش** *(ویندوز؛ اگر از قبل موجود است رد کنید)* — اسکریپت `detect_installed_drivers.ps1` را روی کامپیوتر ویندوزی اجرا کنید:
   ```powershell
   powershell -ExecutionPolicy Bypass -File detect_installed_drivers.ps1
   ```

   این فرمان همهٔ درایورهای امضاشدهٔ PnP (`Win32_PnPSignedDriver`) را شمارش می‌کند و
   `installed_windows_drivers.csv` را می‌نویسد و هر دستگاه را برای لینوکس دسته‌بندی می‌کند.
2. **کپی** `installed_windows_drivers.csv` به پوشهٔ `Migrate to Linux/` روی دستگاه لینوکس (اختیاری — نصب‌کننده سخت‌افزار را به‌صورت زنده هم تشخیص می‌دهد).
3. **نصب روی لینوکس** — از داخل پوشهٔ توزیع، با دسترسی root اجرا کنید:
   ```bash
   cd "Migrate to Linux/Linux Mint (Ubuntu)"
   sudo ./install_device_drivers.sh
   ```

   نصب‌کننده **سخت‌افزار را به‌صورت زنده تشخیص می‌دهد** (lspci/lsusb/lscpu/DMI — منبع
   معتبر، چون نام دستگاه‌های ویندوز به‌طور تمیز به ماژول‌های لینوکس نگاشته نمی‌شود)،
   آن را با CSV تطبیق می‌دهد، سپس فقط آنچه لازم است نصب می‌کند:- **GPU:** درایور اختصاصی NVIDIA (از طریق `ubuntu-drivers`)؛ AMD/Intel درون‌هسته‌ای + Mesa/VA-API.
   - **شبکه:** Realtek `r8168`/`r8125-dkms`، Broadcom `broadcom-sta-dkms`، DKMS وای‌فای USB رئالتک؛ بقیه درون‌هسته‌ای + `linux-firmware`.
   - **CPU:** `intel-microcode` / `amd64-microcode`.
   - **چاپگرها/اسکنرها:** CUPS بدون‌درایور + HPLIP (اچ‌پی) + SANE.
   - **اثر انگشت:** `fprintd` + `libfprint`.
   - **میان‌افزار سازنده:** `fwupd` + LVFS — میان‌افزار UEFI/SSD/داک/جانبی را مستقیماً از Lenovo/Dell/HP/Intel و غیره دانلود و فلش می‌کند.
   - **نصب‌کننده‌های اختیاری سازنده:** فایل `.run` انویدیا (nvidia.com) / AMDGPU-PRO (amd.com) از طریق یک پرسش قابل‌رد‌شدن در انتها.

### چیدمان پوشه‌ها

```text
Migrate to Linux/
├─ detect_installed_windows_software.ps1   # Windows: software inventory + Linux rating
├─ installed_windows_software.csv          # generated software report
├─ additional_Linux_software_requirments.csv # hand-curated: hardcoded apps beyond the CSV
├─ detect_installed_drivers.ps1            # Windows: device-driver inventory + Linux mapping
├─ installed_windows_drivers.csv           # generated driver report
├─ windows_settings_extract.ps1            # Windows: settings scraper
├─ windows_configs.csv                     # generated settings CSV
├─ settings_config.txt                     # human-readable mapping reference
├─ instructions.txt                        # reproducibility spec
├─ README.md
├─ README_fa.md                            # ترجمهٔ فارسی این فایل
└─ Linux Mint (Ubuntu)/                    # one folder per target OS
   ├─ execute_all.sh                       # one-shot: settings → apps → drivers (continue-on-error)
   ├─ install_must_have_software.sh        # unattended software installer
   ├─ install_device_drivers.sh            # unattended driver installer (live detection + fwupd/LVFS)
   └─ apply_settings.sh                    # self-contained settings runner (functions + dispatch + runner)
```

نام هر زیرپوشه برابر با یک **سیستم‌عامل هدف** است؛ برای افزودن هدف جدید یک پوشه بسازید
و اسکریپت‌ها را در آن تولید کنید (به [`instructions.txt`](instructions.txt) نگاه کنید).

## شروع سریع — فهرست‌برداری نرم‌افزار

```powershell
# از داخل پوشهٔ Migrate to Linux/، در PowerShell 5.1 یا PowerShell 7+:
.\detect_installed_windows_software.ps1
```

به دسترسی مدیر (Administrator) نیازی نیست.

### گزینه‌ها

| پارامتر | پیش‌فرض | هدف |
| ------- | ------- | --- |
| `-OutputPath <path>` | `installed_windows_software.csv` (کنار اسکریپت) | محل نوشتن CSV. |
| `-MustIncludeThreshold <int>` | `70` | حداقل *Alternative Competency* (٪) برای **Must be included on Linux = yes**. |
| `-IncludeSystemComponents` | خاموش | نگه‌داشتن بسته‌های توزیع‌مجدد، رانتایم‌ها و درایورها. |
| `-IncludeStoreApps <bool>` | `$true` | شامل‌کردن برنامه‌های فیلترشدهٔ Microsoft Store/UWP. |
| `-Online` | خاموش | پرس‌وجوی زندهٔ repology.org برای برنامه‌های ناشناخته. |

### ستون‌های CSV (فهرست نرم‌افزار)

| ستون | منبع | معنا |
| ---- | ---- | ---- |
| **Name** | از کامپیوتر | نام محصول به‌صورت خوانا. |
| **Version** | از کامپیوتر | نسخهٔ نصب‌شده. |
| **Publisher** | از کامپیوتر | سازنده. |
| **Source** | از کامپیوتر | `Win32` (رجیستری) یا `Store` (UWP/Appx). |
| **Linux Availability** | دستی | پرچم‌ها: Available on Linux، Native Alternative، Available as WebApp و غیره. |
| **Best Linux Alternative** | دستی | بهترین گزینهٔ لینوکس. اگر پولی باشد، جایگزین رایگان هم افزوده می‌شود. |
| **Alternative Competency** | دستی | درصد تقریبی نسبت به ویندوز (≥۱۰۰ یعنی لینوکس بهتر است). |
| **Pricing model** | دستی | Free (FOSS)، Free، Freemium، Shareware یا Paid. |
| **Must be included on Linux** | مشتق‌شده | `yes` / `no` — از روی آستانهٔ شایستگی محاسبه می‌شود. |
| **Can be synched to Linux alternative** | دستی | اینکه آیا دادهٔ برنامه با ورود به حساب (ابری) به‌طور خودکار به جایگزین لینوکسی همگام می‌شود: `Yes` یا `No, manual transfer`. |

### ستون‌های CSV (مهاجرت تنظیمات — `windows_configs.csv`)

| ستون | منبع | معنا |
| ---- | ---- | ---- |
| **Category** | از کامپیوتر | `Power`، `Display`، `Keyboard`، `Telemetry`، `AutoUpdate`، `Screen`. |
| **ConfigKey** | از کامپیوتر | کلید مشخص تنظیم (مثلاً `lid_close_on_ac`، `resolution`، `lock_screen_timeout`). |
| **WindowsValue** | از کامپیوتر | مقدار استخراج‌شده (مثلاً `sleep`، `1920x1080`، `10 min` / `never`). |
| **LinuxCommand** | از کامپیوتر | برچسب‌های زبان ورودی برای نگاشت صفحه‌کلید (اختیاری). |
| **Notes** | از کامپیوتر | یادداشت خوانا دربارهٔ نگاشت لینوکسی. |

### ستون‌های CSV (فهرست درایور — `installed_windows_drivers.csv`)

## رتبه‌بندی‌های لینوکس از کجا می‌آیند

توصیه‌ها **از وب تحقیق شده‌اند** و داخل آرایهٔ `$LinuxKB` در
`detect_installed_windows_software.ps1` جای گرفته‌اند. برای تنظیم یک رتبه یا افزودن
برنامهٔ جدید، آن جدول را ویرایش و دوباره اجرا کنید. حالت اختیاری `-Online` برنامه‌های
*ناشناخته* را از طریق Repology API پر می‌کند.

برای درایورها، نگاشت در جدول قواعد `$DriverKB` در `detect_installed_drivers.ps1` قرار
دارد. هر دستگاه از روی **کلاس PnP** و **شناسهٔ سازندهٔ PCI/USB** نهفته در Hardware ID
دسته‌بندی می‌شود (اولین قاعدهٔ منطبق برنده است). برای رتبه‌بندی دوبارهٔ یک دستگاه یا
افزودن یک تراشهٔ جدید، آن جدول را ویرایش و دوباره اجرا کنید.

## نصب روی لینوکس — نرم‌افزار

```bash
# برای Linux Mint / Ubuntu:
sudo ./"Linux Mint (Ubuntu)/install_must_have_software.sh"
```

نصب‌کننده **بدون نظارت و خودبازگشتی (idempotent)** است، با نمایش پیشرفت تمیز،
تشخیص خودکار دستگاه مقصد (نام رمز توزیع + معماری) و دو فایل لاگ. برای دانلودها یک
**نوار پیشرفت زندهٔ هش** نشان داده می‌شود (`curl -#`). برای نصب‌ها فقط **پیشرفت نصب**
نمایش داده می‌شود (بدون پرحرفی): apt درصد زندهٔ خود را از طریق `APT::Status-Fd`،
flatpak با `--show-progress`، و نصب‌کننده‌های بدون API پیشرفت (Wine، اسکریپت‌های .deb)
به‌جای خروجی پرحرف، یک شمارندهٔ زمان سپری‌شدهٔ غیرتعاملی نشان می‌دهند.

در صورت خطا، اسکریپت **مسیر فایل دانلودشده و فرمان دقیق نصب دستی** را چاپ می‌کند تا
کاربر بتواند دوباره تلاش کند. همهٔ دانلودها به `/opt/migrate-downloads/` می‌روند که با
**مود 0777 و مالکیت کاربر غیر‌root فراخواننده** تنظیم شده تا فایل‌ها قابل‌بررسی یا
نصب دستی باشند.

**پیپرکات (سخت‌کدشده — کلاینت بومی، هرگز جایگزین).** کلاینت لینوکسی PaperCut
(Print Deploy / User Client) توسط **سرور PaperCut خودِ شما** ارائه می‌شود که منبع
دست‌اول و معتبر آن است. متغیر `PAPERCUT_SERVER` (میزبان) و در صورت لزوم `PAPERCUT_PORT`
(پیش‌فرض `9174`) را در بالای اسکریپت — یا به‌صورت متغیر محیطی — تنظیم کنید تا
**بدون نظارت** نصب شود:

```bash
sudo PAPERCUT_SERVER=printdeploy.example.com ./install_must_have_software.sh
```

اگر `PAPERCUT_SERVER` تنظیم نشده باشد (یا دانلود خودکار شکست بخورد)، نصب‌کننده **به
پرسش دستی در انتها واگذار می‌کند** و فایل `.tar.gz` کلاینت لینوکس را که از صفحهٔ کلاینت
سرورتان (`https://<server>:9174`) دانلود شده می‌خواهد. در هر صورت **کلاینت بومی
PaperCut نصب می‌شود — هرگز CUPS جایگزین نمی‌شود**.

در پایان نصب خودکار، اسکریپت کاربر را **ترغیب می‌کند** که فونت‌ها را از یک پوشه وارد
کند. ساز‌و‌کار وارد‌کردن فونت:

1. مسیر پوشه‌ای حاوی فایل‌های فونت را می‌پرسد (از `/dev/tty` خوانده می‌شود).
2. همهٔ فایل‌های فونت آن پوشه را به‌صورت بازگشتی پیدا می‌کند (پشتیبانی از **همهٔ قالب‌های رایج فونت**:
   `.ttf`، `.otf`، `.ttc`، `.woff`، `.woff2`، `.pfa`، `.pfb`، `.afm`، `.pfm`، `.dfont`،
   `.otb`، `.bdf`، `.pcf`، `.gsf`، `.otc`، `.abf`، `.chr`، `.fnt`، `.mxf` —
   پوشش‌دهندهٔ انواع فونت ویندوز، مک و لینوکس).
3. آن‌ها را به `/usr/local/share/fonts/user-import/` کپی می‌کند.
4. `fc-cache` را اجرا می‌کند تا حافظهٔ نهان فونت بازسازی شود و برنامه‌ها (از جمله Wine) آن‌ها را ببینند.
5. نتیجه (OK/SKIP/FAIL) را در لاگ نتایج ثبت می‌کند.

مدیریت خطا:

- پوشهٔ ناموجود یا غیرقابل‌خواندن → پرسش دوباره یا تایپ `skip`.
- پوشه فاقد فایل فونت منطبق → هشدار و پرسش دوباره.
- شکست در کپی یک فایل خاص → از آن فایل بگذر، با بقیه ادامه بده.
- نبودِ tty (محیط غیرتعاملی) → بی‌صدا رد کن.

## نصب روی لینوکس — درایورها

```bash
# برای Linux Mint / Ubuntu:
cd "Linux Mint (Ubuntu)"
sudo ./install_device_drivers.sh
```

با دسترسی root و همان رابط کاربری تمیز، اسپینر و دو فایل لاگ مانند نصب‌کنندهٔ نرم‌افزار
اجرا می‌شود. **سخت‌افزار را زنده تشخیص می‌دهد** و فقط آنچه واقعاً موجود است را نصب
می‌کند؛ پس اجرای آن روی هر دستگاهی ایمن است — سخت‌افزار غایب به‌جای ناموفق، *رد‌شده*
ثبت می‌شود. نکات برجسته:

- **پیش‌فرض درون‌هسته‌ای:** بیشتر دستگاه‌ها (وای‌فای Intel/Atheros/MediaTek، بلوتوث،
  صدا، وب‌کم، NVMe، USB، HID) به چیزی فراتر از هسته + `linux-firmware` نیاز ندارند.
- **نصب فعال در صورت نیاز:** درایور اختصاصی NVIDIA، ماژول‌های DKMS رئالتک/برودکام،
  میکروکد CPU، HPLIP، fprintd.
- **میان‌افزار سازنده:** `fwupd` ریموت **LVFS** را تازه می‌کند و میان‌افزار UEFI/
  SSD/داک را که توسط خود سازندگان بارگذاری و امضا شده فلش می‌کند.
- **آگاه از Secure Boot:** وقتی ثبت MOK برای ماژول‌های DKMS/NVIDIA لازم باشد هشدار می‌دهد.
- **نصب‌کننده‌های اختیاری `.run`/PRO سازنده** در انتها از طریق یک پرسش قابل‌رد‌شدن ارائه می‌شوند.

پس از اجرای درایور **راه‌اندازی دوباره به‌شدت توصیه می‌شود** (ماژول‌های GPU/DKMS،
میکروکد، فلش میان‌افزار).

## ملاحظات

- **شایستگی یک برآورد تقریبی عمدی** برای برنامه‌ریزی است، نه یک بنچمارک.
- رتبه‌بندی‌ها وضعیت چشم‌انداز لینوکس را تا **ژوئن ۲۰۲۶** بازتاب می‌دهند و تغییر خواهند کرد.
- رزولوشن نمایش ممکن است روی سخت‌افزار متفاوت قابل‌بازتولید نباشد.
- اسکریپت‌های هر توزیع (`execute_all.sh`، `install_*`، `apply_settings.sh`) **در پوشهٔ
  هر توزیع** قرار دارند (مثلاً `Linux Mint (Ubuntu)/`) — هر سیستم‌عامل هدف نسخهٔ خودش
  را می‌گیرد. `apply_settings.sh` اکنون یک فایل خودبسندهٔ واحد است (فایل پیشین
  `settings_config.sh` در آن ادغام شده است).
- **درایورها روی لینوکس زنده تشخیص داده می‌شوند**، نه رونویسی از ویندوز: فایل
  `installed_windows_drivers.csv` یک مرجع متقابل است، در حالی که `lspci`/`lsusb`/DMI روی
  خودِ دستگاه معتبرند (نام دستگاه‌های ویندوز به‌طور تمیز به ماژول‌های لینوکس نگاشته نمی‌شود).
